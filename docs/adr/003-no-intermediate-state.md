# ADR-003 No Intermediate State（将军赶路：主干优先、禁止兼容中间态）

## 决策

Spark 的架构演进采用“主干优先”策略：

- **只推进主干大石头**：语义契约冻结、性能/安全门禁、自举闭环。
- **禁止兼容中间态**：不引入长期 alias、双实现、过渡 re-export；接口变更要么一次到位、要么不做。

## 决策依据

- Spark 的目标是“通用通信基础设施”，其最大成本来自 **长期语义分裂**（后端差异、口径不一致、Options 漂移）。
- 过早在细枝末节（微优化/辅助 API）投入，会延迟主干语义冻结，造成后续后端/协议叠加时成倍返工。

## PR Gate（执行约束）

- 新增/修改 public API 时：必须同步更新 contract / docs / examples（至少其一作为证据）。
- 任何“临时兼容层”必须带 **明确删除日期/版本**，且默认关闭；否则拒绝合并。
- 代码结构保持清洁：一个抽象/状态机尽量一个文件；`lib.rs` 只做模块声明与稳定 re-export。

## 影响

- 里程碑节奏会更“硬”：短期可能牺牲一点灵活性，但换来跨平台一致性与长期可维护性。
